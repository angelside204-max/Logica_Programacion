# Juego Piedra, Papel o Tijera

## Introducción al proyecto
Este proyecto corresponde al desarrollo de un juego llamado **Piedra, Papel o Tijera**, realizado como práctica de programación.  
El programa permite que un usuario juegue contra la computadora aplicando reglas simples y lógica condicional.

## Explicación de las principales funcionalidades del código

- El programa muestra un mensaje de bienvenida al iniciar el juego.
- Solicita al usuario que ingrese una opción válida: piedra, papel o tijera.
- Valida la entrada del usuario para evitar opciones incorrectas o numéricas.
- La computadora selecciona una opción de forma aleatoria desde una lista.
- Se comparan ambas elecciones utilizando estructuras condicionales y reglas predefinidas.
- El juego se ejecuta por un número determinado de rondas.
- Se lleva un conteo de puntos del usuario, de la computadora y de empates.
- Se muestran pausas visuales en la ejecución utilizando temporizadores.
- Al finalizar las rondas, se presentan los resultados finales.
- El sistema permite al usuario decidir si desea volver a jugar o finalizar el programa.

## Objetivo del programa
El objetivo del programa es reforzar el uso de condicionales, entrada de datos por teclado y toma de decisiones lógicas dentro de un lenguaje de programación.
